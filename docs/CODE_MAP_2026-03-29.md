# Friction Code Map

Updated after the 2026-03-29 timeline shortcut cleanup.

## Goal

This map is a practical navigation file for future debugging.

It focuses on:

- timeline input routing
- shortcut ownership
- timeline UI expansion state
- where to change behavior without creating more cross-layer coupling

## Current Timeline Shortcut Ownership

### Main global entry

- `src/app/GUI/mainwindow.cpp`

Key functions:

- `MainWindow::eventFilter()`
- `MainWindow::processKeyEvent()`
- `MainWindow::isTimelineInputContext()`

Responsibility:

- detect whether keyboard focus/mouse context belongs to the timeline
- reserve timeline-only keys before they leak into canvas/tool shortcuts
- forward timeline keys to `TimelineDockWidget`

Important rule after this cleanup:

- `MainWindow` only forwards timeline keys to the timeline when the timeline is actually the active input context
- the old fallback that called `mTimeline->processKeyPress()` even outside timeline context was removed

### Timeline command router

- `src/app/GUI/timelinedockwidget.cpp`

Key functions:

- `TimelineDockWidget::processKeyPress()`
- `TimelineDockWidget::currentTimelineWidget()`

Responsibility:

- own timeline-level commands such as:
  - `U`
  - `T`
  - `Ctrl+E`
  - work area / playback related keys
- resolve the active `TimelineWidget`

Important rule:

- if a shortcut is a timeline command and should work regardless of which timeline sub-widget currently has focus, it belongs here

### Timeline local widget behavior

- `src/app/GUI/timelinewidget.cpp`

Key functions:

- `TimelineWidget::eventFilter()`
- `TimelineWidget::handleAeShortcutEvent()`
- `TimelineWidget::showSelectedKeyEaseMenu()`
- `TimelineWidget::showSelectedKeyStrengthMenu()`
- `TimelineWidget::toggleSelectedTransformVisibility()`

Responsibility:

- own timeline-widget-local UI behavior
- still owns `Tab` popup behavior
- delegates specific actions down to keys view or boxes list

Important cleanup:

- `B/N` handling was removed from `TimelineWidget`
- `B/N` now belong to `TimelineDockWidget` only

## Expansion / Reveal State

### Transform row expansion

- `src/app/GUI/BoxesList/boxscrollwidget.cpp`

Key function:

- `BoxScrollWidget::toggleSelectedTransformVisibility()`

Responsibility:

- flip selected layer transform rows open/closed
- work on selected boxes in the active scene

### Row rendering and summary UI

- `src/app/GUI/BoxesList/boxsinglewidget.cpp`

Responsibility:

- render a single layer/property row
- decide what collapsed/expanded row content appears

This is the first place to inspect if a property is logically expanded but the row still looks collapsed.

## Keyframe Menus

### Timeline key selection and easing menus

- `src/app/GUI/keysview.cpp`
- `src/app/GUI/graphboxeslist.cpp`

Key functions:

- `KeysView::showQuickEaseMenu()`
- `KeysView::showQuickStrengthMenu()`
- `KeysView::applyQuickEasingPreset()`
- `KeysView::applyQuickEasingFamily()`
- `KeysView::graphEasingAction()`

Responsibility:

- read currently selected key spans
- apply easing presets to selected or forward segments
- show the actual popup menus used by `Ctrl+E` and `T`

Important dependency:

- quick easing depends on expression presets being available/enabled
- if a menu opens but a preset does nothing, inspect:
  - `src/core/Expressions/expressionpresets.cpp`

## Scene / Timeline Resolution

### Active timeline lookup

- `src/app/GUI/timelinedockwidget.cpp`

Key function:

- `TimelineDockWidget::currentTimelineWidget()`

Responsibility:

- resolve the active timeline page from the stacked timeline layout

This function was simplified during the cleanup and no longer contains temporary debug logging.

## Current Safe Change Rules

Use these rules for future edits:

1. If the bug is ŌĆ£shortcut fires in the wrong placeŌĆØ:
   edit `MainWindow::isTimelineInputContext()` or `TimelineDockWidget::processKeyPress()`.

2. If the bug is ŌĆ£timeline shortcut triggered, but wrong timeline action ranŌĆØ:
   edit `TimelineDockWidget::processKeyPress()`.

3. If the bug is ŌĆ£timeline action ran, but the row/panel did not visually updateŌĆØ:
   inspect `BoxScrollWidget`, `BoxSingleWidget`, and SWT abstraction visibility.

4. If the bug is ŌĆ£easing menu opens but cannot affect keysŌĆØ:
   inspect `KeysView` selection state first, then expression preset enablement.

5. Do not re-add duplicate key ownership to both `TimelineWidget` and `TimelineDockWidget` unless there is a very specific reason.

## What Changed In This Cleanup

- centralized timeline-context detection in `MainWindow::isTimelineInputContext()`
- removed duplicate `B/N` ownership from `TimelineWidget`
- removed non-context fallback routing from `MainWindow::processKeyEvent()`
- reserved `Ctrl+E` correctly in timeline context
- removed temporary timeline debug logs used during the previous investigation
